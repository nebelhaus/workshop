**prowl** is the tiling room. It wraps
[AeroSpace](https://github.com/nikitabobko/AeroSpace) ‚Äî a tiling window manager
for macOS ‚Äî and adds a **Caps-Lock leader** so you can launch, focus, move, and
resize windows without ever reaching for the mouse or a far-flung modifier.

Windows arrange themselves into a tree; the keyboard moves them around it. It
feels like a tiling Linux rig, but it's native macOS windows underneath.

<Image src={tilingGrid} alt="four windows tiled into an even grid ‚Äî a browser, terminal, notes, and chat ‚Äî with the sill bar on top and equal fog-grey gaps between the floating panes" />

## The leader key: Caps-Lock

prowl remaps **Caps-Lock to F18** and uses it as a leader. There are two ways to
use it:

- **Tap ‚á™** (press and release) ‚Üí enter **launch mode**: the bar morphs into
  letter-hint bubbles, and pressing an app's letter launches or focuses it.
- Launch mode is also the gateway to **navigate mode** (arrow keys move focus),
  **resize mode**, workspace jumps, and quick palette actions.

Meanwhile, `‚å•` (Option/Alt) is the everyday modifier for layouts and
workspace-moves.

Focus with *arrows* deliberately isn't a bare `‚å•` chord: `‚å•`, `‚åò`, and `‚åÉ` with
the arrow keys are all macOS text-navigation, so arrow-focus lives behind the
Caps-Lock leader (tap ‚á™, then arrow) where nothing collides. No Vim keys
required ‚Äî but `‚å•H/J/K/L` is there for the hands that want it.

## Everyday tiling ‚Äî main mode

| Keys | Action |
|---|---|
| `‚å•/` | Tiles layout (toggles horizontal ‚Üî vertical split) |
| `‚å•,` | Accordion layout (toggles horizontal ‚Üî vertical) |
| `‚å•F` | Toggle fullscreen |
| `‚å•‚áß1`‚Äì`‚å•‚áß4` | Move window to workspace 1‚Äì4 |
| `‚å•‚áß<app-key>` | Throw the window to that app's home workspace |
| `‚å•Tab` | Jump to the previous workspace (back-and-forth) |
| `‚å•‚áßTab` | Move the workspace to the next monitor |
| `‚å•‚áßR` | Re-sort every window to its workspace |
| `‚å•‚áß;` | Enter **service mode** |
| `‚åòSpace` | Open [Pounce](/guides/pounce/) |

Focusing a workspace is a leader action: tap <kbd>‚á™</kbd>, then a digit (`1`‚Äì`4`)
or an app's letter ‚Äî the same motion whether you're going to scratch space or to
an app.

## Launch mode ‚Äî tap ‚á™

![tap ‚á™ and the bar's workspace pills become letter hints; press an app's letter and it flashes green, launching the app onto its own workspace](/media/tap-caps.webp?v=2)

Tap Caps-Lock, then:

| Key | Action |
|---|---|
| `<app-key>` | Launch or focus that app (e.g. `T` = Ghostty, `B` = Zen) |
| `1`‚Äì`4` | Focus workspace 1‚Äì4 |
| `‚Üê‚Üì‚Üë‚Üí` | Move focus; enters **navigate mode** (see below) |
| `-` / `=` | Enter **resize mode** (shrink / grow) |
| `V` | Clipboard history (via Pounce) |
| `E` | Emoji picker (via Pounce) |
| `Z` | Reopen the last closed app (the ‚åò‚áßT analog) |
| `,` | Open macOS System Settings (mirrors the ‚åò, convention) |
| `/` | Show the cheatsheet |
| `Esc` | Exit launch mode |

**Navigate mode** is how you move focus without Vim keys or a text-nav-clobbering
chord: tap ‚á™, then an arrow moves focus *and* sticks, so further arrows keep
moving. Hold `‚áß` with an arrow to *move* the focused window instead. `Esc` or
`Enter` returns to the desktop.

**Resize mode** is a small delight: once you're in it, you can press `-` or `=`
repeatedly without re-tapping the leader. `Esc` or `Enter` exits.

## Your app roster

The launcher letters, workspace assignments, and the bar's workspace pills all
come from **one list**: `nebelhaus.prowl.apps`. Out of the box it's just a
terminal (`T`) and a browser (`B`). Add your own:

```nix
nebelhaus.prowl.apps = [
  { key = "s"; name = "Slack"; workspace = "S";
    appId = "com.tinyspeck.slackmacgap"; barIcon = ":slack:";
    label = "Slack"; cask = "slack"; }
  { key = "d"; name = "Discord"; workspace = "D";
    appId = "com.hnc.Discord"; barIcon = ":discord:"; cask = "discord"; }
];
```

Each field:

- **`key`** ‚Äî the launcher letter (unique across the roster).
- **`name`** ‚Äî the macOS app name (`open -a` uses it).
- **`workspace`** ‚Äî the AeroSpace workspace letter; `null` means launcher-only,
  no dedicated workspace.
- **`appId`** ‚Äî the bundle id, used to auto-assign the app's windows to its
  workspace. Find it with `osascript -e 'id of app "Slack"'`. `null` skips
  auto-assign.
- **`barIcon`** ‚Äî the [SketchyBar app-font](https://github.com/kvndrsslr/sketchybar-app-font)
  ligature (e.g. `:slack:`) shown in the bar; `null` falls back to the workspace
  letter.
- **`label`** ‚Äî the cheatsheet caption; `null` uses `name`.
- **`cask`** ‚Äî a Homebrew cask to install it; `null` if it's already installed.

Rebuild with `haus rebuild` and your new letters, workspaces, and bar pills all
appear together.

<Aside type="tip" title="Windows untangle after sleep">
macOS loves to scramble window positions when the machine wakes. prowl installs
a wake watcher that re-sorts windows to their workspaces automatically ‚Äî and you
can trigger it yourself any time with `‚å•‚áßR`.
</Aside>

## Turning it off

Don't want tiling? Set `nebelhaus.prowl.enable = false;` and the Caps-Lock
remap, launcher, and AeroSpace all stay out of your way. The rest of the rice
still works.

The complete, always-current binding list ‚Äî including the terminal and Pounce ‚Äî
is on the [Keybindings cheatsheet](/reference/keybindings/).
